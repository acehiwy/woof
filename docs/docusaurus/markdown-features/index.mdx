# Markdown Features

## Standard features

We use [MDX](https://mdxjs.com/) as the parsing engine,
which can do much more than just parsing [standard Markdown syntax](https://daringfireball.net/projects/markdown/syntax),
like rendering React components inside your documents as well.

:::info

MDX is JSX + Markdown

:::

## Front matter

Front matter is used to <Color color="var(--secondary-font-color)"> add metadata to your Markdown file </Color>.
All content <Color color="var(--secondary-font-color)"> plugins </Color> have <Color color="var(--primary-font-color)"> their own </Color> <Color color="var(--secondary-font-color)"> front matter schema </Color>,
and use the front matter to enrich the default metadata inferred from the content or other configuration.

Front matter is provided at the very top of the file, enclosed by three dashes `---`.The content is parsed as [YAML](https://yaml.org/spec/).

```markdown title= showLineNumbers
---
title: My Doc Title
more_data:
  - Can be provided
  - as: objects
    or: arrays
---
```
